Exercise 01.05
如果解释器正则序采用的求值，会得到答案“0”；如果是采用应用序求值，程序会发生死循环。因为如果采用应用序求值的话，程序会计算参数“(p)”，但展开“(p)”时会因为其定义而产生死循环。
而正则序会按如下方式展开：
=> (if (= 0 0) 0 (p))
=> (if #t 0 (p))
此时再展开，就会通过计算条件确定展开“0”还是“(p)”，而依据条件，只会展开“0”，那么结果自然是“0”了。其实，就算是应用序，在展开“(if (= 0 0) 0 (p))”的时候，判断条件后也不会再去展开“(p)”。因为书中明确提到“在求值一个if表达式时，解释器从求值其<predicate>部分开始，如果<predicate>得到真值，解释器就去求值<consequent>并返回其值，否则它就去求值<alternative>并返回其值。
”
